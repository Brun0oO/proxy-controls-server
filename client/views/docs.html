Overview

ProxyControls shares events from input devices (keyboard, gamepad, etc.) with a remote device. While viewing a WebVR site on your phone with Google Cardboard, you can use a USB gamepad — connected to your PC — to move around.

Host Device

For your host (the machine connected directly to the input device), this website is all you need. Open the Start tab to get going:

› Start (Connect as Host Device)

If you run into trouble, please open an issue on GitHub and let me know what’s happening, and how to reproduce the problem.

Remote Device + A-Frame

For your remote viewing device (such as a mobile phone), you’ll need to use a .JS client to receive input events from the host device.

  • A-Frame Proxy Controls Component

In addition to receiving events from the host, you’ll need a controls component to use those events and manipulate the scene. There are currently two official client packages, both designed for A-Frame VR:

  • A-Frame Keyboard Controls keyboard-controls
  • A-Frame Gamepad Controls gamepad-controls
  • A-Frame Leap Motion Controls leap-controls (Coming soon)
  • …or build your own!

After installing proxy-controls and at least one input device controller, use them in your scene: 

<a-scene proxy-controls>
  <a-cube></a-cube>
  <a-entity camera
            gamepad-controls
            keyboard-controls>
  </a-entity>
</a-scene>

Remote Device, without A-Frame

There’s no official pre-packaged support for ProxyControls outside of the A-Frame environment yet. Feel free to build your own, or open an issue on GitHub if you’re interested in discussing an idea.
