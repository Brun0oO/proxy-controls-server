Overview

ProxyControls shares events from input devices (keyboard, gamepad, etc.) with a remote device. While viewing a WebVR site on your phone with Google Cardboard, you can use a USB gamepad — connected to your PC — to move around.

This project works great with Mozilla’s new WebVR library, A-Frame.

Host Device

For your host (the machine connected directly to the input device), this website is all you need:

› Start (Connect as Host Device)

Remote Device + A-Frame VR

Example:

<a-scene proxy-controls>
  <a-cube></a-cube>
  <a-entity camera
            gamepad-controls
            keyboard-controls>
  </a-entity>
</a-scene>

For your remote device (e.g. phone) use a proxy-controls component to receive input events:

  • A-Frame Proxy Controls Component proxy-controls

In addition to receiving events from the host, you’ll need a controls component to apply those events and manipulate the scene. There are a few official client packages, designed for A-Frame:

  • A-Frame Keyboard Controls keyboard-controls
  • A-Frame Gamepad Controls gamepad-controls
  • A-Frame Leap Motion Controls leap-controls (Coming soon)
  • …or build your own!

Remote Device, without A-Frame

There’s no official pre-packaged support for ProxyControls outside of the A-Frame environment yet. Feel free to build your own, or open an issue on GitHub if you’re interested in discussing an idea.
